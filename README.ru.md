# PE Disassembler

![WPF](https://img.shields.io/badge/Platform-WPF-blue)
![C#](https://img.shields.io/badge/Language-C%23-green)
![License](https://img.shields.io/badge/License-MIT-yellow)

Легковесное приложение для Windows, которое дизассемблирует 32-битные PE-файлы (.exe) и отображает инструкции ассемблера из секции `.text`. Создано с использованием WPF и C#.

Английская версия доступна [здесь](README.md)

## Описание
Этот инструмент позволяет исследовать внутреннюю структуру исполняемых файлов, извлекая и декодируя машинный код в человекочитаемые инструкции ассемблера. Он предназначен для образовательных целей и основ реверс-инжиниринга, демонстрируя структуру PE-файлов и кодирование инструкций x86.

## Используемые технологии
- **C#** – основной язык программирования
- **WPF (Windows Presentation Foundation)** – для графического интерфейса
- **.NET Framework / .NET Core** – платформа приложения

## Возможности
- Выбор любого файла и проверка наличия сигнатур MZ/PE.
- Проверка, является ли файл 32-битным исполняемым (x86).
- Извлечение и декодирование ограниченного, но часто встречающегося набора инструкций x86:
  - Перемещение данных: `MOV`, `PUSH`, `POP`
  - Арифметика: `ADD`, `SUB`, `AND`
  - Управление потоком: `JMP`, `CALL`, `RET`, условные переходы
  - Прочие: `NOP`, `CBW`, `CWD`, `PUSHFD` / `POPFD`
- Сохранение результата дизассемблирования в текстовый файл.

## Установка и использование
### Требования
- ОС Windows с .NET Framework 4.7.2+ или .NET 5/6/7/8 (с поддержкой WPF)
- Visual Studio 2019/2022 (для сборки из исходников)

### Шаги по установке
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ddeeduck/Pe-disassembler.git
2. Откройте файл решения (Disassembler.sln) в Visual Studio.

3. Соберите решение (Ctrl+Shift+B).

4. Запустите приложение (F5).

### Как использовать
1. Запустите приложение.

2. Нажмите "Browse" и выберите .exe файл.

3. Нажмите "Check" для проверки структуры PE.

4. Если файл корректен, нажмите "Disassemble" для генерации списка инструкций.

5. Сохраните результат в .txt файл.

### Ограничения
- Поддерживаются только 32-битные (x86) PE-файлы.
- Декодер инструкций покрывает только те опкоды, которые реализованы в исходном коде; многие инструкции пока не распознаются.
- Дизассемблирование выполняется по сырым смещениям в файле, а не по виртуальным адресам (без учёта релокаций).
